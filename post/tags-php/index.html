Tags PHP

Posted on September 14, 2017

Uma coisa bem simples mas que poucas pessoas se atentam são as tags disponíveis para serem usadas no PHP.

Comumente utilizamos a tag “<?php” para iniciar nossos códigos e “?>” para finalizar. Mas existem outras formas de se fazer isso e ainda um alerta para quem costuma fechar a tag PHP.

Possíveis tags para o php:

  1. <?php

    • Este é o padrão recomendado na documentação da linguagem. É o que a maioria dos devs utilizam.

  2. <?

    • Esta forma também é bastante utilizada, principalmente para escreve código php junto ao html de forma mais simples. Porém para utilizar esta forma é necessário habilitar esta opção no php.ini. Por este motivo esta forma não é muito recomendada.

  3. < script language=“php”></ script> e <%

    • Script tags e ASP tags. Estas formas já são bem menos conhecidas e sub utilizadas. Seu uso não é recomendado e foram removidas na versão 7 do php.

O problema das tags php:

Ao escrever um arquivo php, seja ele uma classe, uma função ou um simples formulálrio, não recomenda-se fechar a tag php. Desta forma evita-se escrever espaços em branco ou linhas vazias no final do arquivo, o que iniciaria o buffer de saída quando não tiver nada para enviar.
